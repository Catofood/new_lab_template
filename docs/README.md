# Инструкция по работе с отчетом

## Созданные файлы

1. **[docs/lab.typ](docs/lab.typ)** - Исходный файл отчета в формате Typst
2. **docs/lab.pdf** - Скомпилированный PDF отчет (готов к сдаче)

## Компиляция отчета

### Однократная компиляция

```bash
cd docs
typst compile lab.typ lab.pdf
```

### Режим автообновления (watch mode)

Для автоматической пересборки при изменении файла:

```bash
cd docs
typst watch lab.typ lab.pdf
```

При каждом сохранении lab.typ файл PDF будет автоматически обновляться.

## Добавление скриншотов

### Шаг 1: Создать скриншоты

1. Запустите программу: `.\build\Release\lab1.exe`
2. Сделайте скриншоты работающего приложения:
   - Окно при запуске
   - Вращающийся треугольник
   - Разные положения треугольника

### Шаг 2: Сохранить изображения

Поместите файлы в папку `docs/images/` с именами:
- `Screenshot_1.png` - окно при запуске
- `Screenshot_2.png` - треугольник с градиентом
- `Screenshot_3.png` - треугольник в процессе вращения

### Шаг 3: Раскомментировать код

В файле [docs/lab.typ](docs/lab.typ) найдите раздел "Результаты работы программы" (примерно строки 220-235) и раскомментируйте блоки с изображениями:

```typst
// Было:
// #figure(
//   image("images/Screenshot_1.png", width: 80%),
//   caption: [Окно программы при запуске]
// ) <fig-start>

// Стало:
#figure(
  image("images/Screenshot_1.png", width: 80%),
  caption: [Окно программы при запуске]
) <fig-start>
```

Сделайте то же самое для всех трех рисунков.

### Шаг 4: Пересобрать отчет

```bash
typst compile docs/lab.typ docs/lab.pdf
```

## Структура отчета

Отчет содержит следующие разделы:

1. **Титульный лист** - автоматически сгенерирован
2. **Введение** - цель и задачи работы
3. **Глава 1: Задание на лабораторную работу**
   - Формулировка задания
   - Требования к выполнению
4. **Глава 2: Разработка программы**
   - Выбор среды разработки
   - Структура проекта
   - Конфигурация CMake
   - Описание программы
   - Таблица функций OpenGL
5. **Глава 3: Исходный код программы**
   - Полный листинг main.cpp
6. **Глава 4: Сборка и тестирование**
   - Процесс сборки
   - Результаты тестов (таблица)
7. **Глава 5: Результаты работы программы**
   - Описание визуализации
   - Скриншоты (закомментированы)
8. **Заключение** - итоги работы
9. **Список использованных источников** - 6 источников

## Настройка титульного листа

В файле [docs/lab.typ](docs/lab.typ) в начале найдите секцию `#titlepage` и измените:

- `authors` - ваше ФИО
- `teachers` - ФИО преподавателя
- `position` - должность преподавателя
- `group` - номер вашей группы

```typst
#titlepage(
  title: "ОСВОЕНИЕ СРЕДЫ РАЗРАБОТКИ...",
  authors: ("Фамилия И.О.",),  // ← Измените здесь
  teachers: ("Преподаватель И.О.",),  // ← И здесь
  position: "доцент",
  group: "М411",  // ← Ваша группа
  // ...
)
```

## Форматирование

Отчет автоматически следует стандарту ГОСТ:
- Шрифт: Times New Roman 14pt
- Межстрочный интервал: 1.5
- Выравнивание: по ширине
- Красная строка: 1.25 см
- Поля: стандартные
- Нумерация: автоматическая

**Ничего не нужно настраивать вручную!**

## Проблемы и решения

### Typst не установлен

Установите Typst:
```bash
winget install --id Typst.Typst
```

Или скачайте с https://github.com/typst/typst/releases

### Ошибка компиляции

Проверьте:
1. Правильность путей к файлам (например, `src/main.cpp` существует)
2. Синтаксис Typst (закрывающие скобки, кавычки)
3. Кодировку файла (должна быть UTF-8)

### Изображения не отображаются

1. Убедитесь, что файлы существуют в `docs/images/`
2. Проверьте имена файлов (регистр важен!)
3. Раскомментируйте блоки `#figure`

## Готовность к сдаче

Отчет **готов к сдаче** если:
- ✅ PDF файл создан без ошибок
- ✅ Титульный лист заполнен корректно
- ✅ Исходный код включен (автоматически из main.cpp)
- ✅ Добавлены скриншоты работающей программы
- ✅ Все разделы присутствуют
